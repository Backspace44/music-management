Gestiunea unei Case de Producție Muzicală

Obiectivul Proiectului

Proiectul „Gestiunea unei case de producție muzicală” are ca scop dezvoltarea unei aplicații web ce permite administrarea eficientă a proceselor specifice unei case de producție muzicală. Aplicația permite gestionarea informațiilor despre artiști, albume, piese muzicale, colaboratori și proiecte muzicale, toate integrate într-o platformă centralizată. Scopul principal este de a facilita procesul de urmărire și administrare a producției muzicale de la început până la sfârșit.

Arhitectura Aplicației

1. Roluri, Entități și Procese

Aplicația are patru entități principale:

  • Artiști: Reprezintă persoanele sau grupurile muzicale care creează muzică.
  • Albume: Conțin informații despre albumele muzicale ale artiștilor.
  • Piese muzicale: Sunt piesele individuale din albumele artiștilor.
  • Colaboratori: Persoanele care ajută la producția muzicală, cum ar fi producători, ingineri de sunet, lyriciști etc.
  • Proiecte muzicale: Proiectele la care artistul lucrează împreună cu colaboratorii pentru a crea muzică nouă.

2. Relațiile între Entități

Fiecare artist poate avea unul sau mai multe albume. Fiecare album poate conține mai multe piese muzicale. De asemenea, proiectele muzicale pot implica colaboratori și pot fi asociate cu unul sau mai mulți artiști.

3. Componentele Principale ale Aplicației

Aplicația include două componente principale:

  • Frontend: Folosește HTML, CSS și JavaScript pentru crearea unei interfețe intuitive și interactive.
  • Backend: PHP gestionează logica aplicației, iar MySQL stochează datele în baza de date.

Descrierea Bazei de Date

Baza de date „music_management” conține următoarele tabele:

  • artists: Informații despre artiști (nume, gen, date de contact).
  • albums: Detalii despre albume (titlu, dată lansare, artist_id).
  • songs: Piese muzicale din albume (titlu, durată, album_id).
  • collaborators: Detalii despre colaboratori (nume, rol, date de contact).
  • projects: Proiecte muzicale (nume, descriere, status, artist_id, collaborator_id).
